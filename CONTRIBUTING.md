# Contributing to XAUE

Thanks for your interest in contributing! Here's how to get started.

## Development Workflow

1. Fork the repo
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Make your changes
4. Commit using [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat: add new feature`
   - `fix: resolve bug`
   - `docs: update documentation`
   - `refactor: restructure code`
   - `test: add tests`
   - `chore: maintenance tasks`
5. Push and open a Pull Request

## Branch Naming

| Type | Pattern | Example |
|------|---------|---------|
| Feature | `feat/description` | `feat/token-swap` |
| Bug fix | `fix/description` | `fix/gas-estimation` |
| Docs | `docs/description` | `docs/api-guide` |
| Refactor | `refactor/description` | `refactor/contract-logic` |

## Code Review

- All PRs require at least one review before merging
- Keep PRs focused — one feature or fix per PR
- Write clear commit messages explaining the "why"

## Reporting Issues

- Use the issue templates provided
- Include steps to reproduce for bugs
- Search existing issues before creating new ones

## Security

If you discover a security vulnerability, please **do not** open a public issue. Instead, reach out to us privately.
